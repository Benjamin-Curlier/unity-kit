# Real-world maps as game worlds

## Licensing decides the architecture (verified 2026-07)
- **Google Maps: display-only, never a game world.** The gaming APIs (Maps SDK for Unity, Playable Locations) were decommissioned 2022-12-31. Current ToS ¬ß3.2.3 ban scraping/bulk tile download (elevation included), caching beyond narrow 30-day carve-outs, and "creating content" ‚Äî explicitly tracing roads/building outlines, terrain models from elevation values, point-in-polygon analysis; Map Tiles policy bans geodata extraction and offline use. A baked battlefield derived from Google data is contractually impossible ‚Äî when a user says "use Google Maps", surface this and redirect.
- **Mapbox: same wall** ("No Tracing, Deriving, or Extracting"; 30-day cache cap) and its Unity SDK is abandoned. **ArcGIS Maps SDK for Unity** is alive (local mode = flat plane in meters) but rights to derive redistributable gameplay data from Esri basemaps are unverified ‚Äî check the agreement before depending on it. **Cesium for Unity** is 3D-tiles streaming ‚Äî wrong shape for 2D gameplay grids.
- **The proven path: OpenStreetMap (ODbL) + open DEMs.** The rendered map is a *Produced Work* ‚Üí visible "¬© OpenStreetMap contributors" (link to openstreetmap.org/copyright) on splash/credits. Gameplay grids derived from OSM are a *derivative database* ‚Üí be prepared to publish those grids (never code/art) under ODbL, and pin+archive the source extract for compliance. Precedent: Pok√©mon GO (OSM since 2017), MSFS. **Overture Maps** = better engineering (GeoParquet, stable GERS IDs) but its transportation/buildings/base themes are ODbL like OSM ‚Äî no licensing relief; only its places/addresses themes are permissive.
- **Elevation**: Copernicus DEM GLO-30 (free incl. commercial; mandatory DLR/Airbus credit line; it's a *surface* model ‚Äî compensate canopy/buildings or forests occlude twice), SRTM (public domain, 2000-era), national LiDAR where offered (England OGL 1 m, France Etalab, US 3DEP). **FABDEM is non-commercial ‚Äî never ship it.**

## Pipeline: bake offline, ship bytes
Pinned source extract (Geofabrik region PBF; the public Overpass API is dev-only ‚Äî <10k queries/1 GB per day) ‚Üí filter/classify tags ‚Üí reproject ‚Üí rasterize into (a) render assets and (b) a binary gameplay grid: per-cell movement cost, cover, concealment, blocker flags, elevation, occluder height. The feature‚Üígameplay mapping (road class‚Üíspeed, wood‚Üíconcealment+occluder, building‚Üíblocker/cover/capacity, water‚Üíimpassable unless bridge/ford, DEM‚ÜíLOS) is a design table ‚Äî write it down, make the bake execute it, test it. Tooling: osmium/pyosmium + GDAL/shapely/rasterio in a pinned (Dockerized) offline job; OsmSharp if C# must read PBF. **Runtime does zero geo-processing** ‚Äî load, hash-check, query.

## Projection, scale, precision
Project once, offline, into a **local metric CRS** (UTM zone / national grid / local ENU) centered on the play area ‚Äî 1 Unity unit = 1 m, origin at map center. Never let raw Web Mercator numbers touch gameplay: scale inflates by 1/cos(latitude) (~√ó1.73 at 55¬∞N, ~√ó2 at 60¬∞N). A ~10√ó10 km area centered on origin is float32-safe (‚âà0.5 mm granularity) ‚Äî no floating origin needed at that extent. Geodetic math runs in doubles offline; the engine only ever sees local meters.

## Multiplayer map determinism
The baked grid IS the map truth: content-address it (SHA-256 in a manifest), hash-handshake on join, re-hash at load. Never re-derive gameplay data from source geodata or floats at runtime ‚Äî integer/fixed-point cell records give bit-identical queries on every platform (integer LOS/pathfinding; no float geometry in the sim). Vision stays server-authoritative: LOS computed server-side, so a tampered client map only corrupts its own screen. Map revisions = new hashes; reference the map hash in replays and desync reports.

## Verify
Reproducible bake (pinned inputs + pinned toolchain, recorded hashes; diff two independent bakes before trusting a pipeline change), attribution visible in a screenshot, license credits in the legal screen.
